    Storage Taxonomy

    Tiga model penyimpanan utama di cloud

    Block storage

    Volume mentah untuk VM, database, dan workload yang perlu IOPS stabil.

    EBS, Persistent Disk

    File storage

    Filesystem bersama dengan folder dan permission POSIX/SMB/NFS.

    EFS, Filestore, Azure Files

    Object storage

    Data disimpan sebagai object dengan key, metadata, dan API HTTP.

    R2, S3, GCS, Blob, MinIO
    Object Storage Model

    Object storage bukan filesystem biasa

    bucket
    reports/2026/q1.pdf
    media/avatar.png
    logs/app-0001.jsonl
    • Bucket menjadi namespace global di layanan storage.
    • Key adalah path logis, bukan folder fisik.
    • Metadata menyimpan content-type, etag, size, dan custom tag.
    • API memakai HTTP verbs: PUT, GET, HEAD, DELETE, LIST.
    Cloudflare R2

    Posisi R2 dalam ekosistem cloud

    Cloudflare R2 adalah object storage S3-compatible yang dirancang untuk menyimpan data dekat edge network Cloudflare. Kekuatan utamanya: API kompatibel S3 dan tidak ada biaya egress ke Internet menurut model pricing R2 saat ini.

    Cocok untuk asset aplikasi, backup, data lake ringan, file sharing, log archive, dan origin storage untuk aplikasi web.

    Service Comparison

    R2 dibanding storage sejenis

    Layanan Kekuatan Catatan desain
    Cloudflare R2 S3-compatible, biaya egress Internet tidak dikenakan Bagus untuk app global dan asset delivery
    Amazon S3 Ekosistem paling matang, storage class lengkap Perhatikan biaya egress dan request lintas region
    Google Cloud Storage Integrasi kuat dengan analytics dan Google Cloud IAM dan lifecycle policy perlu dirancang sejak awal
    Azure Blob Integrasi enterprise Microsoft Access tier memengaruhi biaya baca dan penyimpanan
    MinIO Self-hosted S3-compatible Operasional, backup, dan HA menjadi tanggung jawab tim
    Cost Model

    Biaya object storage dibaca sebagai persamaan

    Total = GB-month + Class A ops + Class B ops + egress + fitur ekstra

    Class A: mutasi/list, misalnya PUT, COPY, POST, LIST.

    Class B: baca object, misalnya GET dan HEAD.

    Egress: R2 tidak mengenakan biaya bandwidth keluar Internet pada pricing saat ini.

    Tradeoff: layanan murah di egress bisa tetap mahal bila aplikasi boros operasi LIST.

    Architecture Pattern

    Jangan proxy file besar lewat serverless function

    1 Client minta izin upload

    Browser kirim nama file dan content-type ke API Vercel.

    2 API membuat presigned URL

    Credential R2 tetap di server, URL berlaku singkat.

    3 Browser PUT langsung ke R2

    File tidak melewati memory dan limit request body Vercel.

    4 App menyimpan metadata

    Database opsional menyimpan owner, status, dan audit trail.

    Security Checklist

    Kontrol keamanan minimum

    Reliability

    Object storage kuat, tapi pola akses tetap menentukan

    Key design

    Gunakan prefix bermakna: tenant/date/uuid-name.ext. Hindari key yang mudah ditebak untuk data sensitif.

    Lifecycle

    Atur retensi, penghapusan object sementara, dan tiering bila layanan mendukung.

    Recovery

    Untuk data penting, pertimbangkan versioning, backup bucket, dan replikasi lintas penyedia.

    Observability

    Catat request ID, key prefix, ukuran file, durasi upload, dan error storage.

    Praktikum kelas

    1. Buat bucket R2 dan API token read/write terbatas.
    2. Atur CORS untuk localhost dan domain Vercel.
    3. Isi .env.local pada sample project.
    4. Jalankan npm run dev, upload file kecil, lalu cek object di dashboard R2.
    5. Deploy ke Vercel dan pindahkan variable environment ke dashboard Vercel.
